# Tamkang Mini‑Event Hub


A small, responsive event registration site for the **114 Web Programming Midterm**. It shows upcoming campus events as cards, lets users register with **real‑time validation**, tracks **remaining seats**, and stores registrations in **localStorage**. Includes **dark mode**, **search filter**, and an **export JSON** button.

## Tech stack
- HTML5 semantic layout
- CSS3 + Bootstrap 5 (RWD)
- Vanilla JavaScript (DOM APIs: `querySelector`, `addEventListener`, `createElement`)
- HTML5 Constraint Validation API + custom messages
- `localStorage` for demo persistence


## Features (matches rubric)
1. **Information display platform**: Event cards are generated by JS from a dataset.
2. **Interactive features (2+)**:
- Live **search** (title/tag)
- **Remaining seats** counter updates with selection and edits
- **Password strength** indicator (weak/medium/strong)
- **Export JSON** of registrations
3. **Form validation & UX**:
- Built‑in HTML validation (`required`, `pattern`, `min/max`) with `needs-validation`
- Custom **password policy**: 8–16 chars, include 3 of 4 types (upper/lower/number/symbol)
- Duplicate prevention (same email + event)
4. **Program quality**: Clean file names, concise comments, simple structure
5. **GitHub Pages**: Works as a static site (no server required)
6. **Bonus**: `localStorage`, dark mode, smooth UX animations, full RWD
